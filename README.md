# **Netflix Movies and TV Shows — Data Cleaning**

Пет-проект по очистке данных: датасет Netflix Movies and TV Shows 
с Kaggle содержал пропуски, смешанные типы данных и даты в виде строк. 
Цель - не просто технически убрать пропуски, а для каждой колонки понять 
причину их появления и выбрать обоснованную стратегию обработки.

## **Датасет**

[Netflix Movies and TV Shows](https://www.kaggle.com/datasets/shivamb/netflix-shows) - 
8807 записей, 12 колонок: id, тип контента, название, режиссёр, актёрский 
состав, страна, дата добавления на платформу, год выпуска, рейтинг, 
длительность, жанры, описание.

## **Стек**

Python, Pandas, Jupyter Notebook

## **Что было сделано**

### **Обработка пропусков**

| Колонка | Пропусков | Паттерн | Решение |
|---|---|---|---|
| `director` | 2634 (29.91%) | Сильно связано с типом контента: у Movie пропущено 3%, у TV-Show - 91%. У TV-Show часто нет единого режиссёра | Заполнено `"Unknown"` |
| `cast` | 825 (9.37%) | Слабая связь с типом контента, но чёткий паттерн по жанрам - документалистика, докусериалы, реалити-шоу, детский контент | Заполнено `"Unknown"` |
| `country` | 831 (9.44%) | Структурной причины не выявлено - не связано ни с типом, ни с жанром, ни с другими пропущенными полями | Заполнено `"Unknown"` |
| `date_added` | 10 (0.11%) | Значения нельзя восстановить из других полей | После конвертации в `datetime` было разбито в 3 колонки (`day_added`, `month_added`, `year_added`). Пропуски "закрыты" с помощью `"Unknown"` |
| `rating` | 4 (0.05%) | Точечные пропуски без общего паттерна | Заполнено существующей в данных категорией `"NR"` |
| `duration` | 3 (0.03%) | Пропусков нет, есть сдвиг данных при парсинге исходного файла - значение длительности оказалось записано в `rating` | Значения перенесены в нужную колонку |

### **Исправление типов данных**

- `date_added` → разбита на 3 колонки: число, месяц, год
- `duration` → разбита на числовое значение и единицу измерения

### **Результат**

После очистки в датасете не осталось пропусков и структурных ошибок, 
типы данных приведены к корректным. Дубликатов строк не обнаружено.
